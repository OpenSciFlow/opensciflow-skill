# OpenSciFlow Skill

**OpenSciFlow Skill teaches AI agents how to use OpenSciFlow manifests and workflow templates for safe, local-first, reproducible AI for Science tool execution.**

中文定位：

> OpenSciFlow Skill 让 AI Agent 学会读取 OpenSciFlow 协议，以安全、可检查、可复现的方式调用 AI for Science 工具。

This repository is an early draft. It is meant for correction, not for claiming a mature agent ecosystem.

## Why This Exists

AI for Science already has many strong tools and models, but agents often fail when they try to use them directly from long READMEs, scattered installation notes, issue comments, HPC scripts, and paper supplements.

Common failure points include:

- environment and CUDA conflicts;
- missing model-weight paths and checksums;
- unclear GPU, HPC, or Slurm requirements;
- missing license and citation metadata;
- unsafe shell commands generated by an LLM;
- no reproducible run record after a successful attempt.

OpenSciFlow Skill compresses these tool-use details into a small protocol layer:

- `opensciflow.yaml` plugin manifests;
- workflow templates;
- readiness levels;
- local-agent safety rules;
- run records.

## What This Is

- A skill specification for AI agents.
- A set of prompt templates.
- A set of JSON Schemas.
- A set of safety and refusal rules.
- A set of structured examples.
- A protocol adapter layer for other agents and AI Scientist-style systems.

## What This Is Not

- Not a general AI Scientist.
- Not a replacement for existing scientific tools.
- Not a clinical or drug-development decision system.
- Not permission for agents to execute arbitrary shell commands.
- Not a mature ecosystem or standard body.
- Not a claim of partnership with any listed project.

## Core Flow

```text
user request
-> match workflow template
-> read opensciflow.yaml
-> check readiness / inputs / weights / license / citation / hardware
-> render reviewed command template
-> ask for user approval
-> execute through local agent
-> write run record
```

Most important idea:

> OpenSciFlow Skill reduces the context and trial-and-error cost of AI agents in scientific computing by turning tool-use knowledge into machine-readable manifests and workflow records.

## Repository Contents

- `skill.md`: agent-readable skill instructions.
- `skill.json`: machine-readable skill metadata.
- `docs/`: design notes, adoption guides, safety policy, evaluation plan.
- `schemas/`: JSON Schemas for skill inputs, outputs, workflow plans, execution requests, and run records.
- `prompts/`: prompt templates for planner, manifest reader, workflow matcher, safety checker, command renderer, and run-record writer.
- `examples/`: mock but structured examples for GROMACS RMSD, Slurm/GROMACS RMSD, Slurm/MACE evaluation, DiffDock docking, and Boltz structure prediction.
- `scripts/`: minimal schema validators.
- `tests/`: schema validation and refusal-case tests.

## Key Drafts

- `docs/schema-mapping.md`: how skill input, workflow planning, execution requests, and run records connect.
- `docs/wrapper-review-checklist.md`: when wrapper scripts are allowed and when they must be refused.
- `docs/hpc-slurm-notes.md`: site-specific questions an agent must not invent.
- `docs/refusal-policy.md`: normal fail-closed outcomes.

## Validation

Install the small validation dependencies:

```bash
python -m pip install jsonschema pyyaml pytest
```

Validate example fixtures:

```bash
python scripts/validate_skill_input.py tests/fixtures/valid_skill_input.json
python scripts/validate_skill_output.py examples/gromacs-rmsd/skill-output.example.json
python scripts/validate_execution_request.py examples/slurm-gromacs-rmsd/execution-request.json
python scripts/validate_run_record.py tests/fixtures/valid_run_record.json
```

Run tests:

```bash
python -m pytest
```

## How To Contribute

Good first contributions:

- Review manifest fields.
- Review skill behavior.
- Add refusal cases.
- Add HPC / Slurm missing cases.
- Add examples for scientific tools.
- Correct citations or licenses.
- Report unsafe execution assumptions.

Keep changes small, sourced, and correction-first.
