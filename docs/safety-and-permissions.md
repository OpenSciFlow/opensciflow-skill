# Safety and Permissions

The default policy is fail closed.

## Allowed

- Read local manifest and workflow files.
- Validate schemas.
- Inspect declared inputs and outputs.
- Render reviewed command templates.
- Render reviewed wrapper submission commands when wrapper metadata is present.
- Run dry-run commands after user approval when required.
- Write run records inside approved run directories.

## Not Allowed

- Execute arbitrary shell generated by an LLM.
- Run unreviewed wrapper scripts.
- Download model weights without explicit approval.
- Upload user data by default.
- Write outside approved run directories.
- Invent missing license, citation, or checksum metadata.
- Claim clinical, drug-efficacy, or experimental conclusions.

## Approval Boundary

User approval is required before:

- running commands beyond static validation;
- downloading weights or containers;
- submitting Slurm jobs;
- launching reviewed wrappers that consume GPU, HPC, paid, or shared resources;
- using GPU/HPC resources;
- writing outside the proposed run directory.
